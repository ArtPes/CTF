[1mNAME[0m[m

[1mSYNOPSIS[0m[m

[1mDESCRIPTION[0m[m
       PW-Inspector  reads  passwords in and prints those which meet the requirements.  The return code is the number of valid passwords found, 0 if none was found.  Use for security: check passwords, if 0[m
       is returned, reject password choice.  Use for hacking: trim your dictionary file to the pw requirements of the target.  Usage only allowed for legal purposes.[m

[m
       [1m-m[0m [1mMINLEN[0m[m

              maximum length of a valid password[m
[m
       [1m-c[0m [1mMINSETS[0m[m

       [1m-h,[0m [1m--help[0m[m

[1mNAME[0m[m
       john - a tool to find weak passwords of your users[m
[m
[1mSYNOPSIS[0m[m
       [1mjohn[0m [[4moptions[24m] password-files[m
[m
[1mDESCRIPTION[0m[m
       This  manual page documents briefly the [1mjohn[0m command.  This manual page was written for the De‐[m
       bian GNU/Linux distribution because the original program does not have a  manual  page.   [1mjohn[0m,[m
       better  known  as  John the Ripper, is a tool to find weak passwords of users in a server. John[m
       can use a dictionary or some search pattern as well as a password file to check for  passwords.[m
       John  supports  different  cracking modes and understands many ciphertext formats, like several[m
       DES variants, MD5 and blowfish. It can also be used to extract AFS and Windows NT passwords.[m
[m
[1mUSAGE[0m[m
       To use John, you just need to supply it a password file and the desired options. If no mode  is[m
       specified, john will try "single" first, then "wordlist" and finally "incremental".[m
[m
       Once  John  finds  a  password, it will be printed to the terminal and saved into a file called[m
       ~/.john/john.pot. John will read this file when it restarts so it doesn't try to crack  already[m
       done passwords.[m
[m
       To see the cracked passwords, use[m
[m
       john -show passwd[m
[m
       Important:  do  this  under  the  same directory where the password was cracked (when using the[m
       cronjob, /var/lib/john), otherwise it won't work.[m
[m
       While cracking, you can press any key for status, or Ctrl+C to abort the session, saving  point[m
       information  to  a  file  ( [4m~/.john/john.rec[24m by default). By the way, if you press Ctrl+C twice[m
       John will abort immediately without saving.  The point information is also saved every 10  min‐[m
       utes  (configurable  in the configuration file, [4m~/.john/john.ini[24m or [4m~/.john/john.conf[24m ) in case[m
       of a crash.[m
[m
       To continue an interrupted session, run:[m
[m
       john -restore[m
[m
       Now, you may notice that many accounts have a disabled shell, you can make  John  ignore  these[m
       (assume that shell is called [4m/etc/expired[24m ):[m
[m
       john -show -shells:-/etc/expired passwd[m
